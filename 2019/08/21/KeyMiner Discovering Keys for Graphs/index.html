KeyMiner Discovering Keys for Graphs

  1. 用名字来作为电影的key是不准确的，因为会出现重名的

    as Figure 1 shows two di erent movies titled `Bad Boys’ in 1983 (m1) and in 1995 (m4).

  2. 专业领域的语义会影响key的质量

    比如：电影用name和awards作为key，但是不是所有电影都可以获奖，那么很多电影在awards这一项是空的。

  • 需要在图上自动发现key，并且随着data的加入，需要可以重新定义key。

    随着新数据的加入，有些key会过时。

  • 定义了3种点

    1. wildcard _v：2个点类型一样，但是没有匹配id和value
    2. variable v：2个点类型一样，id和value也一样
    3. value v*： 点只有值，2个点的值相同。

keyMiner

思想：

在图中发现有意义的key依赖于定义与应用程序域和数据无关的key属性，特别是当多个候选键可以识别一个实体时。

paper提出了两个关键的性质：极小性和覆盖性

property1 minimality极小性

key就是用最少的属性定义实体，如果有更多的属性可以加进来也参与，那么这些属性是多余的。

key ：

  1. 有最少的点

  2. 最小数量的实体变量，实体变量会引出递归建，需要更长的匹配时间，wildcards和values的点是可以直接通过节点数量来获取的，不包含递归。

计算key的初始极小度

E1是wildcard和values点的数量，E2是variable点的数量，φ是用户定义的对E1的倾向度，m是用户定义的点数量上限。

min的值在(0,1)，数值越大代表较小的key

property2 coverage 覆盖度

对于候选key Q，coverage代表Q可以捕获的实体数，我们要找可以捕获最多实体的key

key相当于给实体们分类，当然是自己一个类最好

I是Q捕获的实体数量，C是类数，N是实体x在图G中的总实体数。

cov的值在[0,1]，0代表没有实体被捕获，1代表所有实体都被独立的识别。

KeyMiner Algorithm

discoverykey

  1. 生成一个仅包含t类型的x点的pattern

  2. 然后用这个点的邻居们Gdt来扩展（d是hop）

  3. 看这个新候选key是不是递归的

  4. 使用有向无圈图(DAG)跟踪实体之间的依赖关系。如果新加入了一个不是t类型的实体（类型t‘），则在DAG里在点t和点t’之间加条边。如果DAG里有环了，则删除最后添加的s依赖项(t,t’),如果没有环，则递归的在t‘类的点上进行discovery算法

  5. 如果不满足最小示例数，就删掉这个候选key
